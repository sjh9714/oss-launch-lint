# Allowed promotion channels and posting rules

## Default allowed channels
Use only channels the user owns or communities where project sharing is explicitly allowed.

- GitHub profile README and pinned repositories
- Personal X/Twitter, Mastodon, Bluesky, or Threads account
- LinkedIn personal account
- Personal blog or dev.to-style article
- Hacker News Show HN, if the project is genuinely usable
- Relevant Discord/Slack communities where project feedback posts are allowed
- Relevant Reddit communities where self-promotion/project feedback posts are allowed
- Product Hunt or similar launch site, only if the project has a broader audience and the user wants that

## Approval rules
Before posting externally, ask the user for approval with:

- Exact text to post
- Destination/channel
- Whether a login is required
- Any tags/mentions
- Expected risk or rule concern

## Good promotion behavior
- Share the problem and a useful example.
- Ask for feedback, bugs, or missing use cases.
- Mention “star if useful” only as a secondary line, not the main request.
- Reply to comments and fix real issues.
- Post follow-ups only when something meaningful changed.

## Bad promotion behavior
- Do not post the same copy everywhere.
- Do not ask strangers directly to star the repo.
- Do not create fake accounts.
- Do not automate stars, follows, comments, or DMs.
- Do not scrape GitHub users for outreach.
- Do not post in unrelated issues, PRs, discussions, or communities.
- Do not offer rewards, credits, tokens, giveaways, or reciprocal stars.
